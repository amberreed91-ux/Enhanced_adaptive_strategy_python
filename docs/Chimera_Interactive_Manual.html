5) Chimera Strategy Logic (Full Summary)

Layer A: Microstructure Signals (Bookmap Engine)

Engine computes these features from L2 + tape:

  • Imbalance = (bid_total - ask_total) / total
  • Sweep up/down from short burst move + size percentile
  • Absorption from large hit without follow-through
  • Whale event from percentile threshold + minimum size
  • Confidence score capped at 100

Layer B: Decision Votes

Vote logic from Chimera app:

long += 1 if imbalance >= 0.05
short += 1 if imbalance <= -0.05
long += 1 if sweep_up
short += 1 if sweep_down
long += 1 if absorption_bid
short += 1 if absorption_ask
long += 2 if whale_buy
short += 2 if whale_sell

min_conf = 50 (quiet mode) else 40
decision = GO_LONG / GO_SHORT / NO_TRADE

Layer C: Executor Hard Gate

  • Symbol allow-list
  • Daily trade cap + cooldown
  • Max daily loss guard
  • Per-symbol kill switch
  • Session lockout windows
  • Bridge profile checks (strict/balanced/permissive/custom)

Layer D: Journal + Evaluation

  • Orders + closed trades per broker
  • Executor audit trail with reasons
  • A/B profile tracker over rolling trades
  • Backtest + Monte Carlo in app
  • Prop readiness gate panel

ICT Asian Sweep Branch (Range Engine)

  • Uses last completed Asian session high/low.
  • Checks recent sweep beyond range edge by ATR buffer.
  • Requires reclaim back inside range edge.
  • Optionally requires displacement candle and kill-zone time windows.
  • Optional FVG CE and premium/discount filters.
  • Can invert signals with invert_signals=true (currently enabled in combined configs).

Key Formulas

Imbalance_estimate = mean((close - open) / (high - low)) over imbalance_lookback

Confluence =
  int(regime_confidence * 0.7)
  + DRE alignment weight
  + HTF alignment bonus
  + RVOL confirmation bonus
  + structure bonus

Risk size =
  risk_budget_usd / (stop_distance * point_value)
where:
  risk_budget_usd = min(initial_capital * base_risk_percent, max_risk_per_trade_usd)
  stop_distance = max(ATR * regime_stop_mult, min_stop_abs)

Final position size =
  risk_size * regime_multiplier * quality_scale
  then clipped by notional cap, portfolio adjustment, and min/max position size.
Why you sometimes see fewer trades: this stack applies multiple hard filters in sequence (session, quality, regime mode, trend-quality, bridge, risk guard). Lower trade count with better quality is expected behavior.

11) Watchdog and Cooldown Breakdown

        
This is the exact separation of cooldown logic across Chimera app and executor.


        

          
Cooldown / ThresholdDefaultWhereWhat It Does

          
Signal Alert Cooldown15sApp UI setting alert_cooldown_secThrottles BUY/SELL alert spam in app alerts. Does not block execution itself.

          
Watchdog Alert Cooldown (App)120sApp UI setting watchdog_alert_cooldown_secLimits how often the app sends watchdog webhook alerts while errors persist.

          
Order Cooldown (Executor)20sautomation.cooldown_secondsBlocks new fills in executor with reason cooldown_active to prevent rapid-fire entries.

          
Watchdog Phone/Webhook Cooldown (Executor)180sautomation.watchdog.phone_alert_cooldown_secondsLimits outbound phone/webhook watchdog notifications from executor.

          
Feed Stale Threshold20s app / 25s executorApp + configMarks feed stale and raises watchdog alerts if exceeded.

          
Bridge Stale Threshold25s app / 30s executorApp + configFlags stale bridge and can affect readiness/alerts.

          
Executor Idle Threshold180sApp + configFlags idle webhook pipeline if no posts arrive.

        


        

          

            

              
How Watchdog Alert Cooldown Works

              
if watchdog_alerts_exist:
  if webhook_enabled and url_set:
    if (now - last_watchdog_alert_epoch) >= watchdog_alert_cooldown_sec:
      send_watchdog_alert()
      last_watchdog_alert_epoch = now

              
This controls alert frequency only. It does not resume trading or clear the underlying stale condition.

            

          

          

            

              
How Order Cooldown Works (Executor)

              
if time_since_last_trade < automation.cooldown_seconds:
  reject_trade("cooldown_active")
else:
  continue risk checks and execution

              
This one directly blocks execution attempts until cooldown expires.

13) Troubleshooting

        

          
          

            Verify source is updating, bridge path is correct, and executor is running at /status. Reduce stale thresholds only after data is stable.
          

        

        

          
          

            Check webhook URL route, tunnel health, symbol allow-list, and webhook secret. Then inspect data/chimera_execution_audit.jsonl for exact rejection reason.
          

        

        

          
          

            Confirm data CSV has required OHLCV columns, quality gate passes, and strategy thresholds are not too restrictive for selected instrument/session.
          

        

        

          
          

            Inspect confidence and vote spread in Signal Card notes, then use the simulator above to see which feature is missing (imbalance, sweep, absorption, or whale).
